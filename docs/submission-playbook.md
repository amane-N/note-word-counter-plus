### 2.3 詳細説明（Description）

```
note Word Counter+ は、note.com の執筆画面・記事閲覧画面に小さなフローティングウィジェットを追加し、文字数・読了時間・見出し数などをリアルタイムで確認できる無料の執筆支援 Chrome 拡張機能です。

【主な機能】

■ 執筆中のリアルタイムカウント
- 文字数（スペース・改行を除いた実質文字数）
- 読了時間(400字/分を基準に自動計算)
- 見出し数(H1〜H6)
- 段落数
- ハッシュタグ数
- 画像数

■ 目標文字数と進捗バー
ポップアップから目標文字数を設定すると、達成率をプログレスバーで表示します。目標達成時はバーが緑色に変わり、一目で分かります。

■ カスタマイズ可能な表示設定
- ウィジェットの表示位置(右下・左下・右上・左上)
- フォントサイズ(小・中・大)
- 表示する項目を個別にオン／オフ

■ 自由に動かせるウィジェット
ドラッグ操作でウィジェットを好きな位置に移動できます。折りたたみ状態にして最小化することも可能です。

■ 閲覧モード対応
note.com の記事ページでも動作し、他の人の記事の文字数・読了時間をさっと確認できます。

【プライバシーとセキュリティ】
- 外部サーバーへの通信は一切ありません
- 収集するユーザーデータはゼロです
- 設定はブラウザのローカルストレージにのみ保存されます
- 要求する権限は "storage"(設定保存用)と "activeTab" のみです
- note.com と editor.note.com 以外のページでは動作しません

【対応ページ】
- https://note.com/(記事閲覧ページ)
- https://editor.note.com/(執筆ページ)

完全無料でご利用いただけます。
```

## 3. 「プライバシー」タブ（最頻出落とし穴）

### 3.1 単一用途の説明 (Single Purpose)

```
note.com および editor.note.com の執筆・閲覧画面に、文字数・読了時間・見出し数などの執筆メトリクスをリアルタイム表示する。
```

### 3.2 権限の正当化（4つすべて埋める）

**storage**

```
ユーザーが選択したウィジェットの表示位置・フォントサイズ・表示項目・目標文字数の設定値を、ブラウザのローカルストレージ(chrome.storage.local)に保存し、次回起動時に復元するために使用します。外部送信は行いません。
```

**activeTab**

```
現在開いているnote.comのタブ内でコンテンツスクリプトが動作するために必要です。バックグラウンドでの全タブ監視は行いません。
```

**host permission: https://note.com/\***

```
note.comの記事閲覧ページで本文を読み取り、文字数・読了時間を表示するために必要です。
```

**host permission: https://editor.note.com/\***

```
note公式の執筆ページがeditor.note.comサブドメインにリダイレクトされるため、執筆中のリアルタイムカウントを実現するには両ドメインへのアクセスが必須です。
```

### 3.3 リモートコード使用

### 3.6 審査用テスト手順

「テスト用アカウント不要」を選び、テキストフィールドに:

```
1. 拡張機能をインストール後、Chromeで https://note.com/ の任意の記事ページを開いてください。
   → 画面右下にフローティングウィジェットが表示され、その記事の文字数・読了時間が表示されます。

2. https://editor.note.com/ で新規記事作成画面を開いてください。
   → ウィジェットがリアルタイムで文字数を更新します。

3. 拡張機能アイコンをクリックすると設定ポップアップが開き、目標文字数・表示位置・フォントサイズを変更できます。

ログインは不要です。記事ページの公開コンテンツのみで動作確認できます。
```

## よくあるリジェクト理由と対処

| 理由 | 対処 |
|---|---|
| Blue Argon: Permissions | 権限の正当化が不十分。各権限の「なぜ」をより具体的に書き直す |
| Purple Potassium: Privacy Policy | プライバシーポリシーURLが死んでる/不十分。URLを再確認 |
| Spam / Functional issue | スクショと実機能の不一致。スクショを実装に合わせる |
| Single Purpose policy | 詳細説明で機能を盛り込みすぎ。1つの主目的に絞る |
| User Data policy | データ使用の3つの誓約にチェック忘れ |

リジェクトされても**修正版を無制限に再提出**できます。再審査は通常初回より早い。
